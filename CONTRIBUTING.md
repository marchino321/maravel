# Contributing to Maravel Framework

Thanks for your interest in contributing to Maravel!

## Rules
- Keep changes focused and well documented
- Follow existing coding style
- Use clear commit messages
- Open an issue before large changes

## Pull Requests
- Target the `main` branch
- One feature or fix per PR
- Explain *why*, not only *what*

Maravel is actively developed and used in production.
Business-specific plugins and applications are intentionally kept private.
